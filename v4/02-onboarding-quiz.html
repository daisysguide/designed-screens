Question count is dynamic: 4 questions (Trying / thinking about trying) · 5 questions (Expecting again — Q3 auto-set) · 6 questions (Currently pregnant / Baby is already here).

Conditional routing: Q2 (due date picker) only shown when Q1 = "Currently pregnant" OR "Expecting again" · Q2b (baby's birthday picker) shown when Q1 = "Baby is already here" · Q3 skipped entirely when Q1 = "Expecting again" — auto-set to "No" in data layer · Q2 and Q4 both skipped when Q1 = "Trying (or thinking about trying)" (no due date, no baby's-sex question pre-pregnancy)

Back chevron returns to previous question in sequence; from Q1, returns to Screen 01 (Swipeable Intro). Completing all questions → Screen 03. Quiz state is persisted to localStorage (optimistic cache) and to the anonymous Supabase user record (canonical) after each answer; localStorage clears on completion or back past Q1.
All questions + conditional logic
Q1
Where are you in your journey?
Options: Trying (or thinking about trying) · Currently pregnant · Baby is already here · Expecting again · Required — no skip · Determines entire question sequence
Q2
When is your due date?
Native date picker · Min date: today · Label: "Due date" · Shown when Q1 = "Currently pregnant" OR "Expecting again" · Required — no skip
Q2b
When was baby born?
Native date picker · Max date: today · Label: "Baby's birthday" · Shown when Q1 = "Baby is already here" · Used for age-appropriate topic sequencing · Required — no skip
Q3
Will this be your first?
Options: Yes, first time · No, we've done this before · Shown when Q1 = "Trying (or thinking about trying)", "Currently pregnant", or "Baby is already here" · Skipped (auto-set "No") when Q1 = "Expecting again" · Required when shown — no skip
Q4
Have you found out the baby's sex yet?
Options: Yes, it's a girl · Yes, it's a boy · Not yet · We're not finding out · Filters circumcision topic (excluded only for "Yes, it's a girl") · Optional · Skip = treat as "We're not finding out" (circumcision topic included)
Q5
Will you be sharing a last name?
Options: Yes · No · Still deciding · Filters last name topic · Optional · Skip = treat as "Still deciding" (last name topic included)
Q7
How often would you like us to nudge you?
Options: Daily · A few times a week · Weekly · Don't nudge me · Optional · Skip = "A few times a week" · Skip label: "Skip — I'll set this in Settings later"

Anonymous session and data persistence
Anonymous user recordCreated server-side (Supabase anonymous user) on first entry to Screen 02, before Q1 renders. One session per device.
Per-answer writesEvery answer and skip is written to the anonymous user record server-side and mirrored to localStorage. The server record is canonical; localStorage is an optimistic cache for instant resume on app relaunch.
localStorage keydg_onboarding_quiz_v4
localStorage savesAll answers to date + current question index · written after every answer and skip
localStorage resumesOn app relaunch mid-quiz: restore answers from cache, rebuild sequence from Q1 answer, advance to last unanswered question with previous answers pre-selected. Server record is the source of truth if cache is missing or stale.
localStorage clearsOn quiz completion · on back past Q1 (user returns to Screen 01). The anonymous server record persists through these events.
Sign-up mergeOn Screen 04 sign-up, the anonymous record is merged with the new permanent account via Supabase linkIdentity (or equivalent). Quiz answers, anonymous ID, and any partial state attach to the new authenticated user.
Recovery pathA user with an anonymous session and a completed quiz who returns without signing up is routed to Screen 07 (Personalized Results) on next app open — their plan is ready, they just haven't created an account yet.
Retention (unconverted)TBD — flag for Jake. Need a policy for how long unconverted anonymous records are retained server-side before cleanup.

Behavior notes
One question per screen — same template reused for all questions in the sequence.
Q1 answer immediately determines the full sequence and updates the progress bar total. Sequence and progress label recalculate before advancing to the next question.
"Expecting again" silently sets Q3 = "No" in the data layer without showing the question. The user's second-time parent status is inferred.
Q4 circumcision filtering: excluded only when answer is "Yes, it's a girl". All other answers (Yes, it's a boy / Not yet / We're not finding out / skipped) keep the topic included.
Saved state is restored on relaunch — previous answers are pre-selected. User resumes at the first unanswered question.
Tapping a selected option deselects it — Continue returns to disabled state.
Back-nav (within sequence): Tapping the back chevron returns to the previous question in the current sequence. The previously selected answer is pre-selected on the prior question; Continue is enabled.
Back-nav from Q1: Returns to Screen 01 (Swipeable Intro) and clears saved state (localStorage). The anonymous server record persists.
Changing Q1 on back-navigation: Recalculate the sequence from the new Q1 answer. Preserve any answers that exist in both the old and new sequences (e.g., Q5, Q7). Clear Q2/Q2b when switching between a pregnancy stage and "Baby is already here" — the date question changes context (due date vs birthday).
No bottom nav on this screen.
When Q1 = "Expecting again": Q2 (due date) is shown, Q3 is skipped and auto-set to "No" in the data layer. User proceeds from Q2 to Q4.
When Q1 = "Baby is already here": Q2b (baby's birthday) is shown instead of Q2. Q3 is shown normally.
When Q1 = "Trying (or thinking about trying)": Q2 (due date) and Q4 (baby's sex) are both skipped — neither applies pre-pregnancy. Q3 (will this be your first?) is asked. User proceeds Q1 → Q3 → Q5 → Q7.
