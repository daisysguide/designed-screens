Behavior notes
Input auto-focuses on mount — the keyboard should open immediately without any user tap.
CTA is disabled (opacity 0.4) when the input is empty. It becomes active the moment any character is typed.
On tap: trim leading/trailing whitespace before saving. If the result is an empty string, keep the CTA disabled.
Name is saved to the user profile and used as [firstName] throughout the app (e.g., "Now let's get [firstName] in here" on Screen 10).
Apple and Google sign-in users never see this screen. Their display name comes from the auth provider and is saved during the sign-in handshake.
App resume: if the user closes the app on this screen, they resume here with the input empty. This is an addition to the mid-topic-app-resume rules in the design system.
