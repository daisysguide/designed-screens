Three topic states
✓
Included
Filled purple-500 checkbox · full opacity · topic is in the plan
We hid this
Dashed border · opacity: ~0.55 · reason shown below label in text-tertiary · "Tap to add back" in purple-500
You hid this
Empty solid border · opacity: ~0.4 · "You hid this · Tap to add back" below label
Nothing is ever truly gone — every hidden topic has a visible "Tap to add back" affordance. Users are always in control.

Behavior notes
Header count updates live as user toggles topics — "28 topics in your plan" increments/decrements immediately.
Categories are collapsible — tapping the category header collapses/expands the topic rows.
All categories shown — user scrolls through all 7. "5 more below" hint prompts scrolling.
Tapping a "we hid" topic adds it back as included. Tapping an included topic moves it to "you hid".
Sticky bottom CTA label is context-dependent — 'Unlock my plan →' during onboarding, 'Save changes' post-purchase.
When isOnboarding=true: CTA is "Unlock my plan →" → Screen 09 Paywall. User has just invested effort reviewing their plan — highest conversion moment.
When isOnboarding=false: CTA is "Save changes" → returns to the calling screen (Topic List or Settings). isOnboarding is set by the routing layer based on purchase completion state.
No bottom nav in onboarding context. Present in post-onboarding context (accessed from Topic List or Settings).
