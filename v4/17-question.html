Behavior notes
One question per screen. Never show multiple questions simultaneously.
Never auto-advance on selection — user must explicitly tap Next. This app is about considered decisions, not quick taps.
"Read guide" opens Screen 16 Article without losing question progress. Back from article returns to the same question at the same state.
Selection persists if user taps Back and returns to a question. Answers are saved client-side as the user progresses and committed to the server on Submit.
Flow: Q1 → Q2 → Q3 → Q4 → Screen 30 (Note Step) → Screen 18 (Waiting for Partner). Back from Q1 → Screen 15 (Topic Intro). Back from Note Step → Q4.
The Note step is the final step. Submit is always enabled — an empty note is valid and saves as an empty string.
Slider questions: no default value. Continue remains disabled until the user sets a value by tapping or dragging the track.
Multi-select questions: Continue becomes active when at least 1 option is selected.
Partner note text uses the partner's first name from the linked account, or "Your partner" if not yet linked.
No bottom nav on any question or note step screen.
