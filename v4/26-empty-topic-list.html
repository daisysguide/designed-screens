Restoration nudge — deep link behavior
"Finish setting up →" resumes Manage Topics at the last-completed step — not from the beginning. If no steps were completed, starts from the beginning.
Once Manage Topics is completed, navigating back to Topics shows the populated Topic List (Screen 14) normally.
This state is distinct from a filtered-empty Topic List (where all topics in a category are complete). Don't reuse this empty state for filtered results — use inline "No topics here yet" text instead.
Components used
Screen Header Empty State Bottom Nav Button (primary)
Edge case note: This screen should be rare in practice — Manage Topics is a required step in onboarding. But users can reach Topics via the bottom nav at any point, and deep linking from a push notification could also land here if the plan isn't set up. The restoration nudge handles both gracefully.
